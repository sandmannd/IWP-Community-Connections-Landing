IWP COMMUNITY CONNECTIONS LANDING PAGE
======================================

FILES
-----
index.html      Main public landing page
privacy.html    Privacy policy
terms.html      Terms of service
styles.css      Site styling
config.js       Change URLs, support email, and displayed organization details here
site.js         Loads values from config.js

BEFORE PUBLISHING
-----------------
1. Open config.js.
2. Replace PASTE_PUBLIC_APP_URL_HERE with the public Apps Script deployment URL ending in /exec.
3. Confirm the support email, Facebook URL, phone number, and effective date.
4. Purchase or use a custom domain you control.
5. Host these files using a static host such as Cloudflare Pages, Netlify, GitHub Pages with a custom domain, or your domain provider.
6. Connect the custom domain.
7. Verify ownership of the domain in Google Search Console.
8. In Google Auth Platform, add the root domain under Authorized domains.
9. Use:
   Application home page: https://YOURDOMAIN.com/
   Privacy policy:        https://YOURDOMAIN.com/privacy.html
   Terms of service:      https://YOURDOMAIN.com/terms.html

IMPORTANT
---------
Google requires the OAuth homepage to be hosted on a verified domain you own. A free hosting subdomain alone is not the final verification setup. Connect a custom domain before submitting OAuth verification.
